# 俄罗斯方块 (Tetris) 需求规格说明书

## 1. 概述

### 1.1 项目目标
开发一款经典的俄罗斯方块游戏，使用键盘方向键控制方块下落与旋转，提供流畅的游戏体验和计分系统。

### 1.2 目标平台
- 单机桌面应用（Pygame），编程语言使用C++
- 后续可扩展为 Web 版本

### 1.3 核心原则
- 操作响应即时（< 50ms 延迟）
- 画面刷新 60 FPS
- 遵循经典俄罗斯方块的游戏规则

## 3. 方块 (Tetrominoes)

### 3.1 七种标准方块

| 编号 | 名称 | 形状 | 颜色 |
|------|------|------|------|
| I | I 型 (长条) | `####` | 青色 #00FFFF |
| O | O 型 (方块) | `.XX` `XX.` | 黄色 #FFFF00 |
| T | T 型 | `.X.` `XXX` | 紫色 #AA00FF |
| S | S 型 (Z镜像) | `.XX` `XX.` | 绿色 #00FF00 |
| Z | Z 型 | `XX.` `.XX` | 红色 #FF0000 |
| J | J 型 (反L) | `X..` `XXX` | 蓝色 #0000FF |
| L | L 型 | `..X` `XXX` | 橙色 #FF8800 |

### 3.2 旋转系统

采用 **SRS (Super Rotation System)** 旋转规则：

- 所有方块围绕其旋转中心旋转
- 顺时针旋转（↑ 键）
- 支持墙踢 (Wall Kick)：旋转后若与墙壁重叠，自动向左/右偏移

**I 型方块的 4 种旋转状态：**
```
状态0:  ....   状态1:  ..#.   状态2:  ....   状态3:  .#..
        ####          ..#.          ....          .#..
        ....          ..#.          ####          .#..
        ....          ..#.          ....          .#..
```

**O 型方块不旋转**（旋转后形状不变）


## 4. 游戏机制

### 4.1 方块生成
- 使用 7-Bag 随机系统：将 7 种方块各 1 个随机排列入袋，依次取出；袋空后刷新
- 保证不会连续出现同一种方块过多
- 初始方块从顶部中间（第 5-6 列）生成

### 4.2 下落机制
| 参数 | 初始值 | 说明 |
|------|--------|------|
| 下落间隔 | 800ms | 每帧自动下移 1 格 |
| 软降速度 | 50ms | 按住 ↓ 键加速下落 |
| 硬降 | 即时 | 空格键直接落到底部 |
| 锁定延迟 | 500ms | 方块触底后可移动窗口 |

### 4.3 消行规则
- 当一行被完全填满时，该行消除
- 上方所有行整体下移
- 一次最多可消 4 行 (Tetris)

### 4.4 游戏结束条件
- 新生成的方块与已有方块重叠
- 即：顶部 spawn 位置被占用


## 5. 计分系统

### 5.1 消行得分（随等级提高）

| 消行数 | 名称 | 基础分 | 公式 |
|--------|------|--------|------|
| 1 | Single | 100 | 100 × level |
| 2 | Double | 300 | 300 × level |
| 3 | Triple | 500 | 500 × level |
| 4 | Tetris | 800 | 800 × level |

### 5.2 额外加分
| 操作 | 加分 |
|------|------|
| 软降 | 1 分/格 |
| 硬降 | 2 分/格 |

### 5.3 等级系统
- 每消除 10 行升级
- 最高 15 级
- 等级越高，下落速度越快

### 5.4 下落速度表

| 等级 | 间隔 (ms) |
|------|-----------|
| 1 | 800 |
| 2 | 720 |
| 3 | 630 |
| 4 | 550 |
| 5 | 470 |
| 6 | 380 |
| 7 | 300 |
| 8 | 220 |
| 9 | 150 |
| 10+ | 100 |


## 6. 操作控制

| 按键 | 功能 |
|------|------|
| ← | 左移 1 格 |
| → | 右移 1 格 |
| ↓ | 软降（加速下落） |
| ↑ | 顺时针旋转 90° |
| 空格 | 硬降（直接落底） |
| P | 暂停/继续 |
| R | 重新开始 |
| ESC | 退出游戏 |

### 6.1 DAS / ARR（按住连移）
| 参数 | 值 | 说明 |
|------|-----|------|
| DAS (延迟) | 167ms | 按住方向键后开始连续移动的初始延迟 |
| ARR (速率) | 33ms | 连续移动的时间间隔 |

## 8. 视觉与音效

### 8.1 视觉
- 不同方块使用不同颜色填充
- 已固定方块有 3D 边框效果（高亮面+阴影面）
- 消行时闪烁动画（100ms）
- 游戏结束显示 "GAME OVER" + 最终分数
- Ghost Piece（预览方块落点，半透明）

## 9. 技术实现

### 9.1 技术栈
| 层 | 选型 |
|----|------|
| 语言 | Python 3.10+ |
| 图形 | Pygame |
| 构建 | pip + requirements.txt |

### 9.2 项目结构
```
tetris/
├── main.py              # 入口
├── game.py              # 游戏主循环 & 状态机
├── board.py             # 棋盘逻辑
├── piece.py             # 方块定义 & 旋转
├── renderer.py          # 渲染器
├── scorer.py            # 计分 & 等级
├── constants.py         # 常量配置
├── requirements.txt
└── srs.md
```

### 9.3 核心数据结构

```python
# 棋盘：20×10 二维数组，0=空，1-7=方块颜色编号
board: list[list[int]]

# 当前活动方块
active_piece: Piece
    shape: list[list[int]]   # 4×4 旋转矩阵
    x: int                   # 左上角列位置
    y: int                   # 左上角行位置
    type: int                # 方块类型 0-6
    rotation: int            # 旋转状态 0-3
```

### 9.4 游戏循环 (60 FPS)
```
每帧 (16.67ms):
  1. 处理输入事件（键盘）
  2. 更新下落计时器
  3. 检查消行
  4. 更新分数/等级
  5. 渲染画面
  6. 检查游戏结束
```

### 9.5 关键代码（main.py 入口）
```python
import pygame
from game import Game

pygame.init()
game = Game()
game.run()
pygame.quit()
```
